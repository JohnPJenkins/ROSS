3 /*
4  * Grid: the spatial grid represent the space modeled can be created in 3D
5  * by created a 3D cube of LPs where the spacing between LPs in a given
6  * dimension is given by the user, and stored in the array: g_rm_spatial_d.
7  *
8  * The grid is constructed by reading in a file of Z-values given in an XY-plane.
9  * The analogy is that you can think of the g_rm_spatial_terrain_fn file as
10  * a blanket overlaying the terrain. Then each successive 2D layer of the grid
11  * cube is this blanket, but with an offset in the Z-value as specified in
12  * g_rm_spatial_d[2].
13  *
14  * Constructing the grid in this way means that 2 LPs in the LP array may not
15  * be neighbors, because they may contain Z-values > g_rm_spatial_d[2]. Each
16  * Cell LP has pointers to the correct neighbors that allow for the proper traversal
17  * of particles. The pointers to neighbor cells allow us to not recompute the
18  * surrounding cells for each particle that passes through a given cell.
19  *
20  * The grid also *always* defines a bottom-plane, not for cell LPs, but rather
21  * containing the initial z-values for the bottom-most 'blanket'. These can then
22  * be used as offsets in performing location->cell and cell->location lookups.
23  * (g_rm_z_values).
24  */
