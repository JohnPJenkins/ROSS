4 /*
5  * Each RN lp is an instance of a machine which employs the OSI model.
6  * The modules implement the functionality of the OSI model layers which
7  * have been defined in the ./configure step.
8  *
9  * The functional link between the models and the RN core is the rn_lptype.
10  * Each rn_lptype will register functions for the init, event_handler, RC
11  * event handler and final routines. The RN lp will call these functions
12  * but impose on them the OSI model structure. In this way the layers will
13  * be reproduced in the proper order. This will enforce the upstream/downstream
14  * motion of packets within a machine, including de/muxing.
15  *
16  * The RN lp will also be responsible for transmitting lateral rn_messages, which
17  * are defined to be rn_messages between RN machines. When the rn_message is at the
18  * bottom-most layer of the OSI model, the RN lp will then handle the sending of
19  * remote rn_message to another RN lp. This will model the sending of packets between
20  * two machines on a network, thus completing the physical layer of the
21  * OSI model. This should also ensure
22  * that the module development integration is smooth as individual modules should
23  * not be sending packets directly, which is hard for them to do anyway because they
24  * will never know the bottom-most layer type on the remote machine. The RN lp
25  * will not know this either. In fact, it will simply send an RN rn_message to the
26  * other RN lp, and the other RN lp will be responsible for handling that rn_message's
27  * upstream motion on the remote machine.
28  *
29  * This will facilitate the different layering on different types of
30  * machines on the network: hosts, routers, gateways, and future, unforeseen machines.
31  * This will also give the ability for any type of machine to fill in the entire
32  * OSI model as each chooses. For example, a gateway running BGP/TCP would fill
33  * in the appropriate layers, but could fill in additional layers, such as an
34  * experimental application running above the BGP/TCP stack working on the BGP
35  * data.
36  */
