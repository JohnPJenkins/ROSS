341  /*
342  * When the LPs REMOVE timer fires, we should dequeue the next agent
343  * and send it on to it's next location. We must set the next remove
344  * timer appropriately for the agents remaining in our PQ.
345  *
346  * If the remove is from location 0, then this is the first move of the
347  * day, i.e., from home. We activate agent logic to see if agent
348  * will stay home today, due to quarantine, being ill, or having
349  * ill spouse or children to care for.
350  *
351  * NOTE: We want the agent in the event to be consistent with the
352  * agent in the timer, even if they have the same remove timestamp.
353  * The reason for this is that in the reverse computation, I need to put
354  * the agent back into the PQ, and where else can I get it from but the
355  * timer event?
356  *
