36 /**
37  * inter-lp: events which will be sent to a remote lp
38  *
39  * inter-lp events are handled as usual
40  *
41  * intra-lp: events which are sent to another layer on the same lp
42  *
43  * intra-lp events are immediately scheduled and processed, because their
44  * destination is the local machine, and have a zero timestamp offset, which
45  * means the following:
46  *
47  * the next layer is responsible for handling packet data, ie, from TCP
48  * to IP. The IP layer must re-create the TCP packect on the dest
49  * LP.
50  *
51  * abort events: should not let them traverse layers.. first layer to get abort
52  * is then the only one to get rolled back
53  *
54  * NOTE: We should probably base the timestamp off of the network characteristics,
55  * rather than make the model do it. When the model calls this function, and
56  * is the last layer, we should know the proper link to use, and can inform
57  * the model if this event will fail if sent, over the network.
58  */
